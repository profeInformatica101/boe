UD 1: Introducción a la programación en entorno servidor.

El modelo cliente/servidor es una arquitectura de computación distribuida en la que las tareas y cargas de trabajo se dividen entre los proveedores de un recurso o servicio, llamados servidores, y los solicitantes del servicio, llamados clientes. A menudo, los clientes y servidores se comunican a través de una red informática en diferentes máquinas físicas, pero ambos también pueden residir en la misma máquina.

  • Modelo de dos niveles: El cliente se comunica directamente con el servidor. Ejemplo: Cliente de base de datos comunicándose directamente con el servidor de la base de datos.
  • Modelo de tres niveles: Se introduce una capa intermedia llamada capa de lógica de negocio. Ejemplo: Aplicaciones web donde el navegador (cliente) se comunica con el servidor web que a su vez se comunica con el servidor de base de datos.

El servidor web espera solicitudes de los clientes y las procesa generando respuestas. Estas respuestas pueden ser el resultado de:

  • Servir archivos estáticos.
  • Ejecutar scripts o código del lado del servidor, como PHP, Python (Flask, Django), JavaScript (Node.js), etc.
  • Conectar con bases de datos para obtener o manipular datos antes de enviar una respuesta.

A diferencia de las páginas estáticas, que siempre muestran el mismo contenido, las páginas dinámicas muestran contenido basado en parámetros específicos como entradas del usuario, datos de una base de datos, hora del día, entre otros.

  • Tecnologías: PHP, JSP, ASP.NET, Node.js.
  • Bases de datos comunes: MySQL, PostgreSQL, MongoDB.

Los lenguajes de programación en el servidor definen cómo se procesan las solicitudes y cómo se genera el contenido dinámico. Estos pueden variar en función y capacidad.

  • PHP: ampliamente utilizado, especialmente para aplicaciones web.
  • Python: utilizado con frameworks como Django o Flask.
  • JavaScript (Node.js): permite ejecutar JavaScript en el servidor.
  • Ruby on Rails: basado en el lenguaje de programación Ruby.
  • Java: con frameworks como Spring Boot o Java EE.

Los lenguajes de marcas, como HTML, XML o JSON, se integran con lenguajes de programación del servidor para generar contenido dinámico. Esta combinación permite adaptar el contenido basado en los datos procesados en el servidor.

Hay varias tecnologías que apoyan y mejoran la programación en el servidor. Estas incluyen bases de datos, servidores web, servidores de aplicaciones y herramientas de desarrollo.

Un servidor de aplicaciones es un software que aloja y opera aplicaciones para clientes solicitantes. Funciona como un middleware entre las aplicaciones del lado del servidor y el servidor web. Algunos ejemplos populares incluyen Tomcat, WebLogic y WildFly.

La integración con servidores web permite a los servidores de aplicaciones atender peticiones web, procesar la lógica de negocio y devolver respuestas dinámicas. Servidores web populares incluyen Apache, Nginx y Microsoft's IIS.

Existen diversas herramientas que facilitan el desarrollo en el entorno del servidor, desde IDEs (Entornos de Desarrollo Integrado) hasta herramientas de construcción y gestión de dependencias. Algunas populares incluyen Eclipse, IntelliJ IDEA, Maven y Gradle.

Los editores son herramientas que facilitan la escritura de código, proporcionando resaltado de sintaxis, autocompletado y otras funcionalidades. Ejemplos comunes son Visual Studio Code, Sublime Text y Atom. Los compiladores, por otro lado, traducen el código fuente escrito por los desarrolladores a un formato ejecutable. Algunos lenguajes, como Java, requieren compilación antes de la ejecución.

Observaciones para la calificación:

  • Excelente (5): Respuestas precisas, detalladas, y con profundidad que demuestran un entendimiento completo del tema.
  • Bueno (4): Respuestas con algún nivel de detalle, pero con omisiones menores.
  • Aceptable (3): Respuestas generales, carecen de profundidad y puede haber errores menores.
  • Insuficiente (2): Respuestas vagas, con falta de claridad y errores significativos.
  • Deficiente (1): Respuestas incorrectas o sin conocimiento sobre el tema.
