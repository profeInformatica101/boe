Conceptos de Spring Boot

Mantenimiento del Estado

El mantenimiento del estado en aplicaciones web hace referencia a la capacidad de la aplicación para recordar el estado del usuario a medida que navega entre diferentes páginas. Dado que HTTP es un protocolo sin estado, se utilizan mecanismos como sesiones y cookies para recordar el estado del usuario.

Sesiones

Las sesiones en Spring Boot permiten almacenar datos específicos del usuario entre solicitudes HTTP. Cuando un usuario accede a la aplicación, se crea una sesión, la cual puede ser utilizada para almacenar atributos entre diferentes solicitudes.

@RequestMapping("/setSession")
public String setSession(HttpSession session) {
    session.setAttribute("mySessionAttribute", "someValue");
    return "sessionSet";
}

@RequestMapping("/getSession")
public String getSession(HttpSession session, Model model) {
    model.addAttribute("sessionAttribute", session.getAttribute("mySessionAttribute"));
    return "getSession";
}

Autentificación de Usuarios

La autenticación de usuarios es un proceso para verificar la identidad de un usuario. Con Spring Security, es posible manejar varios esquemas de autenticación.

@Configuration
@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
            .antMatchers("/public/**").permitAll()
            .anyRequest().authenticated()
            .and()
            .formLogin();
    }
}

Cookies

Las cookies son datos almacenados en el navegador del usuario y son enviados con cada solicitud al mismo servidor, lo cual las hace útiles para almacenar información de seguimiento entre solicitudes y visitas del usuario.

@RequestMapping("/getCookie")
public String readCookie(@CookieValue(value = "myCookie", defaultValue = "defaultCookie") String myCookie) {
    return "Read Cookie: " + myCookie;
}

Seguridad: Usuarios, Perfiles, Roles

Spring Security es un módulo que proporciona autenticación y autorización, permitiendo de este modo configurar fácilmente la seguridad de tu aplicación. Los roles y perfiles definen lo que un usuario autenticado puede hacer y ver en la aplicación.

Herramientas de Programación

Spring Boot ofrece una variedad de herramientas para ayudar a desarrollar aplicaciones rápidamente. Algunas de las herramientas de desarrollo y pruebas comunes incluyen Spring Initializr para la creación de proyectos, Maven o Gradle para la gestión de dependencias, y Spring Boot DevTools para la recarga automática de cambios en el código.

Pruebas y Depuración
