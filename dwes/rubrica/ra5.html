RA5:Desarrolla aplicaciones web identificando y aplicando mecanismos para separar el código de presentación de la lógica de negocio.

Práctica: Desarrollo de una Aplicación Web con Separación de Lógica y Presentación (MVC)

Descripción de la Práctica

Los estudiantes deben desarrollar una aplicación web utilizando Spring Boot, que muestre claramente una separación entre la lógica de negocio y la capa de presentación. La aplicación debe incluir mecanismos de generación dinámica de interfaz web y mantener el estado de los controles.

Requisitos

  • Desarrollar una aplicación web que contenga al menos una entidad y una interfaz CRUD para interactuar con ella.
  • Utilizar Spring Boot y Thymeleaf para separar la lógica de negocio del código de presentación.
  • La interfaz debe generar dinámicamente formularios y otras estructuras UI en respuesta a eventos de usuario.
  • Documentar y probar el código de manera adecuada.

Etiquetas

Logica de negocio

    • @Controller: Anotación de Spring para indicar que una clase es un controlador.
    • @RequestMapping: Anotación para mapear solicitudes web.
    • @GetMapping, @PostMapping: Anotaciones para mapear solicitudes HTTP GET y POST respectivamente.
    • @ModelAttribute: Anotación que vincula un método parámetro o un método de retorno a un atributo de modelo con nombre.
    • @Autowired: Anotación para inyectar un bean Spring gestionado en un campo.

Capa de presentación

    • th:object, th:action: Atributos Thymeleaf para definir el objeto y la acción de un formulario.
    • th:field: Atributo Thymeleaf para vincular campos del formulario a propiedades de un objeto.
    • th:text, th:value: Atributos Thymeleaf para definir el texto o el valor de un elemento HTML.
    • th:each: Atributo Thymeleaf para iterar sobre colecciones.
    • th:if, th:unless: Atributos Thymeleaf para condiciones.
    • th:switch, th:case: Atributos Thymeleaf para control de flujo (switch-case).
