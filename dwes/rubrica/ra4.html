Práctica: Desarrollo de una Aplicación Web con Spring Boot

Descripción de la Práctica

Los estudiantes se embarcarán en un proyecto práctico para desarrollar una aplicación web utilizando el framework Spring Boot. La aplicación debe ser capaz de gestionar usuarios y sus roles, implementando mecanismos para el manejo de sesiones y cookies, e integrando autenticación de usuarios mediante Spring Security. A su vez, se espera que utilicen herramientas y entornos proporcionados por Spring Boot para facilitar la programación, prueba y depuración del código desarrollado.

La aplicación deberá cumplir con los siguientes requisitos mínimos:

  • Implementación de sesiones para manejar el estado del usuario.
  • Uso de cookies para personalizar la experiencia del usuario en la aplicación.
  • Integración de autenticación de usuarios mediante Spring Security.
  • Desarrollo de la lógica del negocio para gestionar usuarios y sus roles.
  • Uso de herramientas proporcionadas por Spring Boot para desarrollo, prueba y depuración.

Adicionalmente, los estudiantes deberán presentar una documentación técnica que describa el proyecto, la arquitectura utilizada, y justificación de las elecciones técnicas, tecnologías y enfoques utilizados. Se espera que el código fuente y la documentación sean subidos a una plataforma de control de versiones como GitHub para ser evaluados.

Este proyecto práctico servirá como un medio para aplicar los conceptos teóricos aprendidos durante el curso y para desarrollar habilidades prácticas en el desarrollo de aplicaciones web utilizando Spring Boot.

Objetivo

En esta práctica, el objetivo primordial es que el estudiante pueda aplicar los conocimientos teóricos adquiridos sobre desarrollo de aplicaciones web, específicamente usando el framework Spring Boot y tecnologías asociadas como HTML para las vistas, y Spring Security para la gestión de la seguridad y autenticación de usuarios.

Se busca que los estudiantes:

  • Entiendan y apliquen conceptos fundamentales sobre mantenimiento del estado usando sesiones y cookies.
  • Desarrollen habilidades para implementar autenticación y autorización de usuarios a través de Spring Security.
  • Usen herramientas de Spring Boot para facilitar la programación, prueba y depuración del código desarrollado.
  • Aplican conceptos relacionados con la persistencia de datos, si optan por integrar una base de datos en su aplicación.
  • Desarrollen, prueben y depuren aplicaciones web usando las capacidades de Spring Boot.
  • Documenten técnica y adecuadamente el código fuente y la arquitectura de la aplicación.

A través de la creación de una aplicación que cumpla con los requisitos descritos, el estudiante será capaz de demostrar su comprensión y habilidad para aplicar estos conceptos en un contexto práctico y realista.

Requisitos

La aplicación a desarrollar durante la práctica debe cumplir con los siguientes requisitos para asegurar que se han aplicado los conceptos aprendidos durante el curso.

  1. Gestión de Usuarios: La aplicación debe permitir la creación, autenticación y autorización de usuarios, así como asignarles roles específicos.
  2. Manejo de Sesiones y Cookies: Implementar el uso de sesiones y cookies para mantener el estado de la aplicación y personalizar la experiencia del usuario.
  3. Autenticación de Usuarios: Integra mecanismos que aseguren la autenticación de los usuarios al acceder a las distintas secciones de la aplicación.
  4. Herramientas y Entornos de Desarrollo: Utilizar herramientas que faciliten la programación, prueba y depuración del código, tales como Spring Boot DevTools y ambientes de desarrollo integrado (IDE) adecuados.
  5. Documentación Técnica: Incluir documentación que describa la arquitectura de la aplicación, las tecnologías utilizadas, y las instrucciones para su despliegue y uso.

Entregables

Los estudiantes deberán entregar los siguientes componentes, asegurándose de que cada uno de ellos cumpla con los requisitos y expectativas establecidas para esta práctica.

  1. Código Fuente de la Aplicación: Presentar todo el código fuente desarrollado, incluyendo archivos de configuración, modelos, controladores, vistas y cualquier otro componente de código utilizado en el proyecto.
  2. Documentación Técnica: Incluir un documento que contenga:
    • Descripción del Proyecto: Una descripción general de la aplicación, explicando su propósito, funcionalidades y público objetivo.
    • Arquitectura Utilizada: Describir la arquitectura del software, incluyendo las tecnologías y protocolos utilizados.
    • Justificación Técnica: Explicar las razones detrás de las elecciones técnicas, incluyendo por qué se optó por ciertas tecnologías, patrones de diseño o metodologías.
    • Instrucciones de Despliegue: Proveer un paso a paso detallado para que otros desarrolladores puedan desplegar y probar la aplicación en un entorno local o de producción.

Es crucial que los entregables sean presentados de manera ordenada y clara, asegurando que cualquier persona con conocimientos técnicos pueda entender, desplegar y usar la aplicación. La documentación técnica debe ser exhaustiva y servir como guía detallada sobre el proyecto y el código fuente.

Dependencias de Spring Boot

Para desarrollar la aplicación en Spring Boot, es necesario agregar diversas dependencias que facilitarán la implementación de las funcionalidades requeridas, como la gestión de usuarios, seguridad, persistencia de datos, y más. A continuación se enlistan las dependencias básicas que podrán ser utilizadas en este proyecto.

  • Spring Web: Para el desarrollo de aplicaciones web y la construcción de API RESTful.
  • Spring Security: A fin de manejar la autenticación y autorización dentro de la aplicación.
  • Thymeleaf: Utilizado como motor de plantillas para renderizar las vistas HTML.
  • Spring Data JPA: Facilita la interacción con la base de datos a través de la API de Persistencia Java (JPA).
  • H2 Database/MySQL/PostgreSQL: Dependiendo de las preferencias y requerimientos, alguna de estas bases de datos puede ser implementada en el proyecto.
  • Spring Boot DevTools: Proporciona herramientas de desarrollo útiles, como la recarga automática de cambios en el código.
  • Spring Boot Test: Ofrece utilidades para realizar pruebas en las aplicaciones de Spring Boot.

Es importante revisar las versiones actuales de las dependencias en la página oficial de Spring Initializr para asegurar la compatibilidad y aprovechar las últimas características y correcciones de seguridad.
