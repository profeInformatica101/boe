RA9. Desarrolla aplicaciones web híbridas seleccionando y utilizando tecnologías, frameworks servidor y repositorios heterogéneos de información.

Práctica: Desarrollo de una Aplicación Web Híbridas

Desarrollar una aplicación web híbrida que cumpla con los siguientes requisitos:

  • Utilizar Spring Boot y microservicios.
  • Integrar diferentes microservicios como configserver, eureka server, etc.
  • Implementar pruebas unitarias y de integración con Spring Boot.
  • Crear y procesar repositorios de información, ya sea utilizando información ya existlibrerías y frameworks para añadir funcionalidades específicas a la aplicación web.ente o creando repositorios específicos.
  • Utilizar
  • Programar servicios y aplicaciones web reutilizando código e información generados por terceros.
  • Realizar pruebas, depuración y documentación de las aplicaciones generadas.

Rúbrica de Evaluación

Criterio Insuficiente Suficiente Bien Notable Sobresaliente
a) Reutilización de código No reutiliza código ni aprovecha información existente. Reutiliza código e información de manera básica y sin optimización. Reutiliza código y aprovecha información con algunas optimizaciones. Reutiliza código e información eficientemente en la mayoría del proyecto. Reutiliza código e información de manera óptima a través de todo el proyecto.
b) Identificación de tecnologías y frameworks No identifica tecnologías y frameworks adecuados. Identifica parcialmente tecnologías y frameworks básicos. Identifica tecnologías y frameworks con algunas justificaciones válidas. Identifica y justifica adecuadamente la mayoría de las tecnologías y frameworks utilizados. Identifica y justifica de manera excepcional todas las tecnologías y frameworks utilizados.
c) Creación de aplicación web No crea una aplicación web funcional. La aplicación web tiene funcionalidad limitada y cumple con pocos requisitos. La aplicación web es funcional y cumple con la mayoría de los requisitos. La aplicación web es altamente funcional y cumple con casi todos los requisitos. La aplicación web es plenamente funcional y cumple con todos los requisitos de manera excepcional.
d) Creación de repositorios específicos No crea repositorios específicos. Crea repositorios básicos con poca o ninguna personalización. Crea repositorios con algunas personalizaciones y funcionalidades. Crea repositorios bien estructurados y con funcionalidades relevantes. Crea repositorios altamente optimizados, estructurados y con funcionalidades avanzadas.
e) Utilización de librerías y frameworks No utiliza librerías ni frameworks. Utiliza librerías y frameworks de forma básica y limitada. Utiliza librerías y frameworks con coherencia y alguna optimización. Integra librerías y frameworks de manera eficiente y coherente. Utiliza librerías y frameworks de manera óptima, innovadora y eficaz.
f) Programación con información y código de terceros No utiliza código ni información de terceros. Utiliza código e información de terceros sin integración efectiva. Integra código e información de terceros con algunos errores o problemas. Integra código e información de terceros de manera efectiva y sin errores notables. Integra código e información de terceros de manera impecable y altamente eficaz.
g) Uso de librerías para Big Data e inteligencia de negocios No utiliza librerías de Big Data ni inteligencia de negocios. Utiliza librerías de Big Data e inteligencia de negocios básicamente. Utiliza librerías de Big Data e inteligencia de negocios con aplicaciones prácticas. Implementa librerías de Big Data e inteligencia de negocios de forma efectiva y justificada. Implementa librerías de Big Data e inteligencia de negocios de forma óptima y con aplicaciones avanzadas.
h) Pruebas, depuración y documentación No realiza pruebas, depuración ni documentación adecuada. Realiza pruebas, depuración y documentación básicas y limitadas. Realiza pruebas y depuración con documentación aceptable. Realiza pruebas exhaustivas y depuración con buena documentación. Realiza pruebas y depuración exhaustivas con documentación excepcional.
