Práctica: Creación y Documentación de un Servicio Web con Spring Boot

Objetivos:

  • Desarrollar un servicio web reutilizable y accesible mediante protocolos web usando Spring Boot.
  • Documentar apropiadamente el servicio web.

Requisitos:

  • Conocimientos básicos en Java y Spring Boot.
  • Conocimientos básicos en el uso de herramientas de prueba de API como Postman.
  • Conocimientos en documentación de API (Swagger/OpenAPI).

Descripción de la Práctica:

  1. Creación del Servicio Web:

    Desarrollar un servicio web utilizando Spring Boot que permita gestionar entidades básicas como "Usuarios". Debe proporcionar operaciones CRUD (Crear, Leer, Actualizar, Eliminar).

  2. Documentación del Servicio Web:

    Utilizar Swagger para documentar el servicio web desarrollado, asegurándose de que cada endpoint esté debidamente descrito y que la documentación esté accesible a través de una URL.

  3. Verificación y Consumo del Servicio Web:

    Utilizar herramientas como Postman para verificar el funcionamiento del servicio web. Crear un cliente básico (puede ser con otro Spring Boot o utilizando tecnología front-end) para consumir el servicio web y demostrar su funcionalidad.
